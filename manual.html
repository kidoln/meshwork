User Manual

MeshWork 操作手册

MeshWork 是围绕笔记、讨论、AI 与 Git 共享构建的创作工作台。它的使用方式不是先创建一个大型在线团队空间， 而是先建立自己的私有知识库，再按需加入共享空间，与团队和 AI 一起把讨论沉淀为长期知识。

01

快速开始

首次打开应用

  1. 选择或创建一个私有知识库。这里存放你自己的长期笔记、草稿、资料和未公开想法。
  2. 进入主工作台。默认会先打开私有知识库，因为它是最稳定的个人入口。
  3. 需要团队协作时，再创建或导入共享空间。共享空间可以稍后再配置 Git。
不需要一开始就配置远程仓库。MeshWork 的设计原则是先让你开始写作和整理，再逐步打开共享能力。
02

私有知识库与共享空间

私有知识库

私有知识库属于当前用户自己，不从属于任何团队或项目。适合保存个人笔记、研究资料、写作草稿、灵感片段和不应自动进入团队共享的内容。

  • 默认长期存在
  • 可以不启用远程 Git
  • 可作为多个共享空间的私有参考素材

共享空间

共享空间是团队或项目的数据边界。每个共享空间包含公共知识库、讨论区和共享附件，也可以拥有自己的 Git 历史。

  • 公共知识库沉淀团队文档
  • 讨论区围绕材料进行异步协作
  • 每个共享空间独立配置同步边界
03

讨论与 AI 协作

从材料发起讨论

你可以围绕一个文档、一个片段或一个主题创建讨论。讨论更接近论坛或邮件列表的节奏，适合深度思考，而不是追求即时聊天的速度。

提出问题把待讨论的材料、背景和目标放进话题。
邀请 AI让 AI 总结、补充视角、分析分歧或提出下一步。
沉淀结论把确认后的共识保存回公共知识库或个人笔记。

AI 在 MeshWork 中是参与者，而不是自动替代创作者的主体。它可以提出建议，但不会未经确认直接修改你的文档。

操作细节：当你在讨论中 @ 一个智能体并让它回答后，接下来的两分钟内，你继续发送的消息不需要再次 @ 它。这个智能体会自动跟进并回复，适合连续追问、补充背景或要求它展开上一轮回答。
04

智能对话命令系统

三种对话增强命令

MeshWork 的讨论区支持三类预定义命令，让对话更高效、与知识库连接更紧密。

@ 提及命令

使用 @ 符号可以提及智能体或其他对话参与者，让他们参与到当前对话中。

@claude 邀请 Claude 智能体参与对话
@张三 提及团队成员张三
提及智能体后，两分钟内的后续消息会自动发送给该智能体，无需重复 @。

# 文件引用

使用 # 符号可以直接引用知识库中的文档内容，让 AI 能够基于具体材料进行分析和讨论。

#产品设计 引用"产品设计"文档
#会议记录/2024-01 引用子目录中的文档
引用文件时，支持自动补全。输入 # 后会弹出匹配的文件列表供选择。

/ 自定义命令

使用 / 符号触发预定义的自定义命令，快速执行常见操作或调用特定功能。

/summarize 总结当前讨论内容
/action 提取待办事项
/template 使用预设模板回复
自定义命令可以在设置中配置，创建符合团队工作流程的快捷操作。

组合使用技巧

三种命令可以组合使用，创建更丰富的对话场景：

  • @claude 请分析 #产品需求 文档中的技术可行性
  • @产品经理 #设计稿 #原型 基于 #竞品分析 给出建议
  • /summarize 并 @全体成员 确认下一步行动计划
04

Git 与 GitHub 共享

把 Git 当成隐藏的可靠底座

MeshWork 使用 Git 承载版本历史、修改追踪、协作共享和恢复能力。你不需要在日常使用中理解 commit、branch、merge 等概念。

  1. 个人内容保存在私有知识库自己的边界内。
  2. 团队内容保存在对应共享空间自己的边界内。
  3. 如果需要通过 GitHub 分享，再为对应资源配置远程仓库。
  4. 共享前确认哪些内容进入公共知识库，哪些仍留在私有知识库。

用远端分支区分不同知识库

MeshWork 允许同类资源共用同一个远端仓库，但不会把多个知识库写进同一条历史线。每个私有知识库或共享空间都会绑定到这个远端仓库中的一个独立分支，用分支来隔离不同知识库的数据。

同类资源可以共用远端 多个私有知识库可以使用同一个远端仓库，多个共享空间也可以使用同一个远端仓库，只要它们使用不同分支。
每个资源对应一个分支 分支留空时，MeshWork 会根据资源类型、名称和 ID 自动生成分支名，避免不同知识库互相覆盖。
不同类型资源不混用 一个远端仓库如果已经被私有知识库使用，就不会再作为共享空间的绑定目标；反过来也一样。

这意味着你可以把一个远端仓库理解成“某一类知识库的远端容器”，而具体的知识库由分支区分。同步时，MeshWork 只会拉取和推送当前资源绑定的那个分支。

工作台本身不是 Git 仓库。Git 只绑定真实数据边界：私有知识库，或某一个共享空间。远端仓库负责承载这些边界，分支负责区分同类资源下的不同知识库。
05

推荐工作流

收集资料 整理笔记 发起讨论 AI 辅助分析 确认结论 沉淀为文档

对个人创作者来说，MeshWork 可以作为长期知识库和写作工作室。对小团队来说，它更适合产品策划、内容创作、研究讨论、世界观设定和项目复盘。
